FAQ

Common Questions

Every project starts with a discovery session covering business goals, user needs, and technical constraints. I move through research, information architecture, wireframing, high-fidelity UI, prototyping, and usability testing — all with WCAG compliance baked in — before a clean developer handoff. The full 16-step framework is outlined in the Process section above.
I establish baseline metrics before any redesign — conversion rates, bounce rates, task completion, time-on-page. Post-launch, I recommend A/B testing key layouts and tracking changes against the baseline. Typical client results include 30–200% conversion improvement and 25–40% reduction in drop-off rates.
Accessibility is designed in, not bolted on. All colour choices pass WCAG 2.1 AA contrast ratios (4.5:1 minimum for text). I annotate wireframes with semantic HTML structure, ARIA labels, focus order, and keyboard navigation patterns. Every handoff includes an accessibility checklist and automated Axe/Lighthouse audit results.
A UX Audit: 1–2 weeks. Full Product Redesign: 4–8 weeks depending on scope. Design Partnership retainers run on 2-week sprint cycles with weekly design delivery.
Figma for all UI design, prototyping, and design systems. Framer for high-fidelity motion prototypes, Spline 3D for 3D elements, Maze and UserTesting for research, Adobe Creative Suite for graphic work. Handoffs include Figma Inspect documentation, design tokens, and accessibility annotations.
